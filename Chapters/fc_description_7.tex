This chapter describes how \ac{Fatori-V} turns user-defined \texttt{.yaml} run files into configured experiments.  It covers the run infrastructure, how the platform sets up the Py2HWSW layer described in \Cref{chap6:fatori_hardware}, benchmark porting, metric collection, and result-table generation. It also presents the Pblock Sizing Model, which estimates module pblock sizes so configuration memory injections can be pointed to selected hardware targets.
