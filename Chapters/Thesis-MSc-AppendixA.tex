\chapter{Appendix A: Fatori-V Run File}
\label{app_a:yaml}

This appendix shows a compact example of a \ac{Fatori-V} \texttt{.yaml} run file, used to setup a full experiment. The file is organized in three top-level sections: identification (used for folder naming and organization), general (where the major knobs reside), and specifics (where the user configures what was enabled in the general section).

\begingroup
\setlength{\columnsep}{28pt}     % more separation between columns
% \setlength{\columnseprule}{0.1pt} % optional: thin vertical rule between columns
\begin{multicols}{2}
\raggedcolumns

Identification Section\par
\begin{lstlisting}[language=simpleyaml,style=yamlcompact,breaklines=true,aboveskip=4pt,belowskip=6pt]
run: # Top-level run descriptor
  identification: # Naming + stamping + reproducibility
    name: "example" # Output folder/prefix
    description: "Example, simple Ibex"
    seed: 5432 # Global seed
    run: fpga 
  hardware:
    board: xcku040
\end{lstlisting}

        target_pool_size: 10000000
        ratio: 100 # % of Register Injections

          mode: random

          module_mode: round_robin

            alu: on

    reg_m_of_n: # Register-level M-of-N redundancy

        dummy_instr: on

      target:

        controller: on

        alu: off
        multiplier: off

    self-testing: # Periodic self-test scheduling
      policy: abort
      spacing: 10
      target:
        load_store_unit: off

        alu: off
        multiplier: off

  metrics: # Optional metrics level overrides
    ibex_hpmc_num: null
    ibex_hpmc_width: null
